## A

- **Agente**: sistema de IA que recibe una tarea y la ejecuta (no solo sugiere).
- **AGENTS.md**: archivo de memoria del proyecto que Codex lee en cada task.
- **Aprobación (modo)**: política para aplicar cambios (`auto`/`ask`/`manual`).
- **API key**: credencial para usar Codex desde CLI / SDK / CI.
- **Apply (codex apply)**: aplicar un patch generado por Codex.
- **Approval mode**: ver "aprobación".

## B

- **Branch base**: rama desde la que Codex parte para abrir un PR.
- **Bug repro**: test que demuestra un bug antes de arreglarlo.

## C

- **Caching**: reutilización de dependencias o contexto entre tasks.
- **Capítulo**: cada sección de la guía (este libro tiene 25).
- **CLI**: la herramienta `codex` para terminal.
- **Cloud (Codex Cloud)**: sandbox en infraestructura de OpenAI para tasks largas.
- **Codebase**: el código fuente completo de tu proyecto.
- **Code review**: revisar cambios antes de mergear.
- **Codex (2021)**: modelo viejo, NO es esto.
- **Codex (actual)**: el agente de coding de OpenAI.
- **Concurrencia**: tasks corriendo en paralelo.
- **Context window**: cuánto texto procesa el modelo de una vez.
- **Copilot**: asistente de GitHub con foco en autocompletar.
- **CSR**: server-side rendering.

## D

- **Delegar**: pasar una tarea a Codex para que la haga sin tu supervisión activa.
- **Dependency**: librería externa de la que depende tu código.
- **Desktop app**: la app de ChatGPT para macOS/Windows.
- **Diff**: cambio entre dos versiones de un archivo.
- **Dictado**: input por voz.
- **Drag-drop**: arrastrar archivos a la app.

## E

- **Edge Function**: función serverless de respuesta rápida.
- **Env var**: variable de entorno.
- **Enterprise**: plan empresarial de ChatGPT con controles avanzados.
- **Exec (codex exec)**: ejecución no interactiva.

## F

- **Fase (de refactor)**: división de un cambio grande en partes manejables.
- **Fastify**: framework HTTP alternativo a Express.
- **Fix**: arreglo de un bug.
- **Flag**: opción en línea de comandos (`--flag`).
- **Frontmatter**: metadata YAML al inicio de un Markdown.

## G

- **GitHub**: plataforma de hosting de código + PRs.
- **GitHub Actions**: CI integrado en GitHub.
- **gpt-5-codex**: familia de modelos de Codex.

## H

- **HIPAA**: regulación de salud en US.
- **Honeypot**: campo invisible para detectar bots.
- **Hot corner**: esquina activa de pantalla.
- **HTTPS**: protocolo seguro de web.
- **Hybrid (Astro)**: mezcla estática + on-demand.

## I

- **IDE**: entorno de desarrollo (VS Code, JetBrains).
- **Inline**: sugerencia que aparece mientras tipeás.
- **Issue (GitHub)**: ticket de bug o feature.

## J

- **JetBrains**: familia de IDEs (IntelliJ, PyCharm, WebStorm).
- **JSON-LD**: datos estructurados para SEO.

## L

- **Lint**: herramienta que detecta problemas de estilo/calidad.
- **Lockfile**: archivo que fija versiones exactas de dependencias.
- **Local (task)**: corre en tu máquina, no en Cloud.

## M

- **MCP**: Model Context Protocol, estándar para conectar IA con herramientas.
- **Mini (modelo)**: el más barato y rápido.
- **Mock**: simulación de una dependencia en tests.
- **Móvil (Codex)**: app de iOS/Android.

## N

- **Networking (sandbox)**: configuración de red para tasks Cloud.
- **NPM**: gestor de paquetes de Node.

## O

- **OAuth**: estándar de autorización.
- **Opt-out training**: desactivar uso de tu data para entrenar modelos.
- **Override**: forzar un valor por encima del default.

## P

- **Pagefind**: librería de búsqueda estática.
- **Patch**: archivo de diff aplicable.
- **PR (Pull Request)**: propuesta de cambio en GitHub.
- **Prompt**: instrucción en lenguaje natural.

## R

- **Race condition**: bug por orden de ejecución concurrente.
- **Rate limit**: tope de requests por tiempo.
- **Reasoner (modelo)**: modelo que razona más antes de responder.
- **Refactor**: cambio de estructura sin cambiar comportamiento.
- **REPL**: read-eval-print loop, sesión interactiva.
- **Repo**: repositorio de código.
- **Repro**: ver "bug repro".
- **Resume**: reanudar una sesión.
- **Reviewer**: quien revisa un PR.
- **RLS**: Row Level Security (Supabase).

## S

- **SAML**: estándar de SSO.
- **Sandbox**: entorno aislado de ejecución.
- **SDK**: librería para usar el servicio desde código.
- **Secret**: dato sensible (API keys, passwords).
- **Setup script**: `.codex/setup.sh`, se corre antes de la task.
- **Slash command**: comando en chat tipo `/codex`.
- **SOC 2**: certificación de seguridad.
- **SSO**: single sign-on.
- **Stripe**: plataforma de pagos.
- **Sync**: sincronización entre superficies.

## T

- **Task**: unidad de trabajo en Codex.
- **Task pane**: panel lateral con tasks.
- **TDD**: test-driven development.
- **Test**: código que verifica comportamiento.
- **Thinking (mode)**: razonamiento extendido del modelo.
- **Token**: unidad de medida de input/output (~4 letras en promedio).
- **Tool**: herramienta externa que Codex puede llamar.

## U

- **UI**: interfaz de usuario.

## V

- **Vercel**: plataforma de hosting (donde está esta web).
- **VS Code**: editor de código de Microsoft.

## W

- **Webhook**: callback HTTP a una URL tuya cuando pasa un evento.
- **Workflow**: secuencia de pasos en CI.
- **Workspace**: ámbito de un proyecto.

## X / Y / Z

- **YAML**: lenguaje para frontmatter y config.
- **Zod**: validador de schemas en TypeScript.
