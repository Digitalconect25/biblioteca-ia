Lovable construye aplicaciones y sitios web full-stack conversando con la IA. Le describís la idea (o le pasás una referencia visual), la IA arma un prototipo funcional en tiempo real y vos lo refinás charlando hasta dejarlo listo para publicar. Sirve para quien quiere pasar de una idea a una app que funciona sin escribir código a mano, incluyendo la parte de datos y usuarios, no solo la pantalla.

## Cómo empezar

1. Entrá a [lovable.dev](https://lovable.dev) y creá tu cuenta.
2. Empezá desde un chat en blanco o desde una plantilla (portfolios, blogs, e-commerce, eventos).
3. Describí la app que querés. Podés sumar una referencia visual para guiar el diseño.
4. Mirá el prototipo que arma en vivo y pedile ajustes conversando ("agregá login", "que los productos se guarden en una base de datos").
5. Conectá integraciones si las necesitás y publicá con un click.

## Ejemplo real

Un prompt para arrancar una app concreta:

```
Quiero una app para gestionar turnos de una peluquería.
- Pantalla pública: lista de servicios con precio y un formulario para
  pedir turno (nombre, teléfono, servicio, día y hora).
- Panel de admin con login: ver, confirmar y cancelar turnos del día.
- Guardá los turnos en una base de datos.
Estilo limpio y moderno, en español.
```

Después iterás: "agregá que al confirmar un turno se envíe un mail al cliente".

## Trucos y errores comunes

- Separá el pedido en funciones claras (pantallas, datos, usuarios): la IA arma mejor cuando entiende cada parte.
- Subir una referencia visual ayuda muchísimo a que el diseño salga parecido a lo que tenés en la cabeza.
- Probá cada cambio en la vista previa antes de seguir pidiendo cosas nuevas, así no se acumulan errores.
- La capa gratuita tiene límites de uso; tareas grandes los consumen rápido.

## Cuándo conviene

Conviene cuando necesitás una app completa, no solo una pantalla: con datos, login y lógica de negocio, sin querer programar todo a mano. Es buena opción para validar una idea o lanzar un MVP rápido. Si solo necesitás UI o componentes React para tu propio proyecto, v0 es más directo; si preferís ver y tocar el código en el navegador como en un editor, mirá Bolt.new.
