v0 es la herramienta de Vercel para construir interfaces y aplicaciones web conversando con la IA. Le decís en texto qué querés y te devuelve componentes React listos (con Next.js y Tailwind), una vista previa en vivo y la opción de publicarlo al instante. Sirve para quien necesita una UI o una landing rápido sin arrancar de cero, y para quien después quiere llevarse el código a su propio proyecto.

## Cómo empezar

1. Entrá a [v0.app](https://v0.app) e iniciá sesión con tu cuenta de Vercel.
2. Abrí un chat nuevo ("New Chat") o partí de una plantilla del catálogo (landings, dashboards, apps).
3. Escribí en lenguaje natural lo que querés. Cuanto más específico, mejor sale.
4. Mirá la vista previa, pedile cambios conversando o ajustá en el modo diseño (colores, tipografía, texto).
5. Cuando te guste, publicá con un click (deploy en Vercel) o exportá el código a GitHub para seguir trabajando.

## Ejemplo real

Un prompt para arrancar una landing concreta:

```
Creá una landing page para una cafetería de especialidad llamada "Grano".
Secciones: hero con foto de fondo y un botón "Reservá tu mesa", lista de 3
productos destacados con precio, un formulario de reserva (nombre, fecha,
cantidad de personas) y un footer con dirección y redes. Estilo cálido,
tonos tierra, tipografía moderna. Usá React y Tailwind.
```

Después iterás: "hacé el hero más alto y agregá un menú de navegación fijo arriba".

## Trucos y errores comunes

- Pedí una sección o pantalla por vez: prompts gigantes salen peor que iterar de a poco.
- Si vas a integrar el resultado en un proyecto propio, exportá a GitHub en vez de copiar y pegar suelto.
- Los créditos gratuitos se consumen por generación; prompts complejos gastan más rápido.
- Revisá el código generado antes de mandarlo a producción: la IA acierta mucho, pero no todo.

## Cuándo conviene

Conviene cuando trabajás (o querés trabajar) en el stack de Vercel: React, Next.js, Tailwind y deploy en Vercel. Es ideal para prototipar UI rápido, armar landings o partir de una base sólida que después editás vos. Si necesitás un backend complejo y multiusuario desde el día uno, una herramienta más full-stack como Lovable o Bolt.new puede encajar mejor.
